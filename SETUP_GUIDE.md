# Final Setup Guide

To get your bot fully running in production, complete these 3 steps:

## 1. Set up Supabase Database
Go to your [Supabase Dashboard](https://supabase.com/dashboard), open the **SQL Editor**, and run this script to create the necessary table:

```sql
create table trades (
  id bigint generated by default as identity primary key,
  tx_hash text not null,
  input_mint text,
  output_mint text,
  amount_in bigint, -- stored in lamports/raw units
  status text default 'confirmed',
  timestamp timestamptz default now()
);
```

## 2. Configure Environment Variables
You need to set these "Secrets" in your hosting provider (Vercel/Railway) and your local `.env` files.

**Required Variables:**
- `PRIVATE_KEY`: Your Solana Wallet Private Key (Base58 string).
- `SUPABASE_URL`: Found in Supabase Settings -> API.
- `SUPABASE_KEY`: Found in Supabase Settings -> API (service_role or anon key).

## 3. Deploy
### Frontend (Dashboard) -> **Vercel**
1. Import your GitHub repo `Resteral/bot` into Vercel.
2. **Important:** Set the "Root Directory" to `meme-bot-dashboard`.
3. Add environment variables if you add authentication later (currently not needed for public view).
4. Deploy!

### Backend (Bot Engine) -> **Railway / Heroku / VPS**
*The bot needs to run 24/7 to watch prices, so it cannot be hosted on Vercel (which shuts down when not used).*

**Option A: Railway (Easiest)**
1. Create a New Project -> Deploy from standard GitHub repo.
2. Select `Resteral/bot`.
3. Set "Root Directory" to `meme-bot-engine` in Settings.
4. Add your `PRIVATE_KEY`, `SUPABASE_URL`, and `SUPABASE_KEY` variables.
5. It will auto-detect `npm start` and run your bot!

**Option B: Run Locally**
Just keep your terminal open:
```bash
cd meme-bot-engine
npm start
```
